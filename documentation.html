Architecture & Overview

SecureShare is a state-of-the-art secure platform that supports zero-disk buffer AES-256 client/server encrypted file sharing. The system integrates standard database schemas with real-time updates and fully asynchronous SMTP notifications.

Real-Time Sync via Server-Sent Events (SSE)

Rather than continuously querying the database every 4 seconds, the system establishes a long-running, unidirectional connection from the client to the server using the HTML5 EventSource protocol. When stats change, updates are pushed instantly.

Express Endpoint 
// sseRoute in routes/files.js
router.get('/sse', authMiddleware, (req, res) => {
  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');
  res.setHeader('Connection', 'keep-alive');

  const clientId = Date.now();
  const newClient = { id: clientId, userId: req.user._id, res };
  sseClients.push(newClient);

  // Broadcast helper to sync changes
  res.write(`data: ${JSON.stringify({ type: 'CONNECTED' })}\n\n`);

  req.on('close', () => {
    sseClients = sseClients.filter(c => c.id !== clientId);
  });
});

Brevo HTTP REST Email API

Because cloud environments like Render restrict outgoing SMTP ports (25, 587, 465) on free instances, we bypassed this network filter by making standard, secure HTTP POST calls to Brevo's Transactional Email API.

Nodemailer to HTTP REST Transfomer 
const sendDownloadAlert = async (uploaderEmail, fileName, timestamp, country) => {
  const senderEmail = process.env.SENDER_EMAIL || process.env.SMTP_USER;
  
  // Connects to Brevo API over standard HTTPS Port 443
  const response = await fetch('https://api.brevo.com/v3/smtp/email', {
    method: 'POST',
    headers: {
      'api-key': process.env.SMTP_PASS,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      sender: { name: "SecureShare", email: senderEmail },
      to: [{ email: uploaderEmail }],
      subject: `📥 File Downloaded: ${fileName}`,
      htmlContent: htmlTemplate
    })
  });
};

Geolocation & Location Mapping

Upon file request, the backend extracts the download client's outbound IP address. Using ip-api.com, it decodes the region parameters and transforms the country code into flag emojis.

Flag Resolver Helper 
const getFlagEmoji = (countryCode) => {
  if (!countryCode) return '🌐';
  const codePoints = countryCode
    .toUpperCase()
    .split('')
    .map(char => 127397 + char.charCodeAt(0));
  return String.fromCodePoint(...codePoints);
};

Vercel Single-Page App (SPA) Routing

To support standard React push state navigation without causing Vercel to look up file pages locally and throw 404 errors, we defined explicit rewrites in the project config root.

vercel.json 
{
  "rewrites": [
    {
      "source": "/api/(.*)",
      "destination": "/api/index.js"
    },
    {
      "source": "/((?!api).*)",
      "destination": "/index.html"
    }
  ]
}

Environment Configuration Parameters

To run the application, configure the following key-value pairs in your environment variables:

Variable Key Service Purpose
VITE_API_URL Vercel (Frontend) The live address of the backend server (e.g. https://secureshare-1der.onrender.com).
MONGO_URI Render (Backend) MongoDB connection string.
JWT_SECRET Render (Backend) Private key used to sign client authorization tokens.
GOOGLE_CLIENT_ID Render & Vercel Google OAuth Client ID.
SMTP_USER Render (Backend) Brevo login username (e.g. af7685001@smtp-brevo.com).
SMTP_PASS Render (Backend) Brevo REST API key / password (starts with xkeysib-).
SENDER_EMAIL Render (Backend) Your verified Brevo sender email address (e.g. p.shreyambbk@gmail.com).
